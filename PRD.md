# PRD: 볼링 동호회 회비 관리 앱

**문서 버전**: 1.0
**작성일**: 2026-01-12
**상태**: 초안 (Draft)

---

## 1. 프로젝트 개요

### 1.1 배경

볼링 동호회(약 20명)에서 현재 엑셀로 관리 중인 회비, 장부, 점수 기록을 모바일 앱으로 전환하여 총무의 관리 편의성을 높이고 데이터의 일관성을 확보한다.

### 1.2 목표

| 목표 | 측정 지표 |
|------|----------|
| 회비 관리 효율화 | 납부 기록 시간 50% 단축 |
| 실시간 점수 기록 | 볼링장에서 즉시 입력 가능 |
| 투명한 장부 관리 | 수입/지출 내역 즉시 조회 |
| 정확한 핸디캡 산출 | 자동 계산으로 오류 제거 |

### 1.3 범위

**포함 (In Scope)**
- 회원 정보 관리 (CRUD)
- 회비 납부 내역 관리
- 수입/지출 장부 관리
- 볼링 점수 기록 및 통계
- 핸디캡 자동 계산
- 팀전 관리 기능
- 랭킹 시스템

**제외 (Out of Scope)**
- 온라인 결제 연동
- 자동 알림/푸시 기능 (v2 고려)
- 회원용 조회 앱 (v2 고려)
- iOS 지원 (v2 고려)

## 2. 사용자 정의

### 2.1 주 사용자: 총무

| 항목 | 내용 |
|------|------|
| 역할 | 동호회 총무 (1명) |
| 기술 수준 | 일반 스마트폰 사용자 |
| 사용 환경 | 볼링장 현장, 자택 |
| 주요 니즈 | 빠른 데이터 입력, 한눈에 현황 파악 |

### 2.2 사용 시나리오

**시나리오 1: 정기 모임일**
1. 볼링장 도착 후 참석 회원 확인
2. 게임 진행하며 점수 실시간 입력
3. 모임 종료 후 레인비/식비 지출 기록
4. 잔액 확인

**시나리오 2: 월초 회비 수금**
1. 이번 달 회비 납부 화면 열기
2. 납부한 회원 체크박스 선택
3. 일괄 납부 등록
4. 미납자 목록 확인

**시나리오 3: 월말 정산**
1. 월간 수입/지출 내역 조회
2. 미납자 목록 확인
3. 회원별 에버리지 및 랭킹 확인
4. 필요시 별도 비용 청구 메모

### 3.3 모임비 정산

게임비와 식비는 회비(공금)에서 먼저 지출하고, 모임 종료 후 참석자별로 n분의 1 정산합니다.

#### 3.3.1 정산 프로세스
```
1. 모임 진행 중 총무가 게임비/식비를 회비에서 지출
2. 모임 종료 후 총 지출 금액 확인
3. 참석 인원으로 나누어 1인당 부담금 계산
4. 회원별 청구 금액 생성
5. 회원들에게 개별 청구 (카카오톡 등)
6. 입금 확인 후 수금 처리
```

#### 3.3.2 정산 등록
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| 모임일 | 날짜 | O | 정산 대상 모임 |
| 참석자 | 다중선택 | O | 정산 대상 회원들 |
| 게임비 | 숫자 | O | 레인비 총액 |
| 식비 | 숫자 | X | 회식비 총액 |
| 기타비용 | 숫자 | X | 기타 비용 |
| 메모 | 텍스트 | X | 정산 내용 설명 |

#### 3.3.3 자동 계산
```
총 비용 = 게임비 + 식비 + 기타비용
1인당 부담금 = 총 비용 ÷ 참석 인원 (원 단위 반올림)
```

**예시**
| 항목 | 금액 |
|------|------|
| 게임비 (5레인) | 85,000원 |
| 식비 | 120,000원 |
| 총 비용 | 205,000원 |
| 참석 인원 | 12명 |
| **1인당 부담금** | **17,100원** |

#### 3.3.4 정산 현황 조회
- 모임별 정산 내역
- 회원별 미수금 목록 (아직 안 낸 사람)
- 회원별 수금 완료 체크
- 정산 완료/미완료 상태 표시

#### 3.3.5 정산 화면 UI
```
┌─────────────────────────────────┐
│  📋 1/12 모임 정산               │
├─────────────────────────────────┤
│  총 비용: 205,000원              │
│  참석: 12명 | 1인당: 17,100원    │
├─────────────────────────────────┤
│  수금 현황 (10/12명 완료)         │
│  ☑ 김철수    17,100원  ✓ 완료   │
│  ☑ 이영희    17,100원  ✓ 완료   │
│  ☑ 박민수    17,100원  ⏳ 미수금  │
│  ☑ 최지훈    17,100원  ⏳ 미수금  │
│  ...                            │
├─────────────────────────────────┤
│  [청구 메시지 복사]  [정산 완료]   │
└─────────────────────────────────┘
```

#### 3.4.4 영수증 OCR 스캔
카메라로 영수증을 촬영하면 자동으로 금액, 날짜, 상호명을 인식하여 입력합니다.

**스캔 프로세스**
```
1. 장부 등록 화면에서 [📷 영수증 스캔] 버튼 탭
2. 카메라로 영수증 촬영
3. OCR로 텍스트 자동 인식
4. 인식된 정보 확인 및 수정
5. 장부에 자동 등록
```

**인식 항목**
| 항목 | 설명 |
|------|------|
| 금액 | 결제 금액 (숫자 + "원" 패턴) |
| 날짜 | 거래 일자 |
| 상호명 | 볼링장/식당 이름 → 내용 필드에 자동 입력 |

**영수증 스캔 UI**
```

├─────────────────────────────────┤

│                                 │

└─────────────────────────────────┘

        ↓ 인식 완료

┌─────────────────────────────────┐
│  ✅ 인식 결과                    │
```

#### 3.4.5 장부 조회

- 수입/지출 구분 필터링
- **잔액 표시**: 상단에 현재 잔액 항상 표시

#### 3.4.6 통계

- 카테고리별 지출 비율 (원형 차트)
- 월별 추이 그래프

### 3.5 찬조 관리

회원 또는 외부인이 동호회에 금액이나 물품을 찬조할 때 기록합니다.

#### 3.5.1 찬조 유형
| 유형 | 설명 | 예시 |
|------|------|------|
| 금액 찬조 | 현금 또는 계좌이체 | 10만원 찬조 |
| 물품 찬조 | 경품, 용품 등 물품 | 볼링공, 상품권 등 |

#### 3.5.2 찬조 등록
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| 찬조자 | 텍스트 | O | 찬조자 이름 (회원/외부인) |
| 유형 | 선택 | O | 금액/물품 |
| 금액 | 숫자 | △ | 금액 찬조 시 필수 |
| 물품명 | 텍스트 | △ | 물품 찬조 시 필수 |
| 물품 수량 | 숫자 | X | 물품 개수 (기본값 1) |
| 추정가액 | 숫자 | X | 물품의 추정 금액 |
| 날짜 | 날짜 | O | 찬조 받은 날짜 |
| 용도 | 텍스트 | X | 사용 용도 (예: 1월 대회 경품) |
| 메모 | 텍스트 | X | 기타 참고사항 |

#### 3.5.3 찬조 현황 조회
- 전체 찬조 내역 리스트
- 기간별/유형별 필터링
- 찬조자별 내역 조회
- 물품 찬조 사용/미사용 상태 관리

#### 3.5.4 물품 찬조 상태
| 상태 | 설명 |
|------|------|

| 사용완료 | 경품 등으로 지급 완료 |

### 3.7 핸디캡 시스템

#### 3.7.1 핸디캡 계산 방식

**수동 입력 방식 (채택)**

총무가 회원별로 핸디캡을 직접 설정합니다. 주로 여성 회원에게 +10 ~ +20점 핸디캡을 부여하여 실력 차이를 보정합니다.

```
핸디캡 점수 = 실제 점수 + 개인 핸디캡
```

**예시**
| 회원 | 성별 | 에버리지 | 핸디캡 | 게임 점수 | 핸디캡 적용 점수 |
|------|------|----------|--------|-----------|------------------|
| 김철수 | 남 | 180 | 0 | 175 | 175 |
| 이영희 | 여 | 150 | +15 | 145 | 160 |
| 박민정 | 여 | 140 | +20 | 135 | 155 |
| 최지훈 | 남 | 170 | +5 | 165 | 170 |

#### 3.7.2 회원 정보에 핸디캡 필드 추가
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| 핸디캡 | 숫자 | X | 개인 핸디캡 (기본값 0, 범위 0~30) |

#### 3.7.3 핸디캡 설정 옵션
| 설정 항목 | 기본값 | 설명 |
|-----------|--------|------|
| 최소 핸디캡 | 0 | 핸디캡 하한선 |
| 최대 핸디캡 | 30 | 핸디캡 상한선 |
| 기본 핸디캡 | 0 | 신규 회원 기본값 |

### 3.8 팀전 기능

#### 3.8.1 팀 구성
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| 팀명 | 텍스트 | O | 팀 이름 |
| 팀원 | 다중선택 | O | 팀에 속한 회원들 |
| 팀 색상 | 색상 | X | 구분용 색상 |

#### 3.8.2 팀전 대회 등록
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| 대회명 | 텍스트 | O | 팀전 대회 이름 |
| 날짜 | 날짜 | O | 대회 날짜 |
| 참가 팀 | 다중선택 | O | 참가하는 팀들 |
| 게임 수 | 숫자 | O | 팀전 게임 수 |
| 핸디캡 적용 | 스위치 | O | 핸디캡 적용 여부 |

#### 3.8.3 팀전 점수 계산
**스크래치 방식**
```
팀 점수 = 팀원들의 실제 점수 합계
```

**핸디캡 방식**
```
팀 점수 = 팀원들의 (실제 점수 + 개인 핸디캡) 합계
```

#### 3.8.4 팀전 결과 조회
- 대회별 팀 순위
- 팀별 상세 점수 (팀원 개인 점수 포함)
- 팀전 히스토리
